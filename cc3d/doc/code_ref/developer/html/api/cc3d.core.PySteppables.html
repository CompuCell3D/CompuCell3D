cell_velocity(_cell: cc3d.cpp.CompuCell.CellG) → cc3d.cpp.CompuCell.Coordinates3DDouble¶

Get the velocity of a cell, in units lattice sites / step

Note that this method is slightly slower than manually calculating velocity from cell attributes but is safe for periodic boundary conditions.

To manually perform the same calculations, do something like the following,

cell: CompuCell.CellG
vx, vy, vz = cell.xCOM - cell.xCOMPrev, cell.yCOM - cell.yCOMPrev, cell.zCOM - cell.zCOMPrev
cell_velocity = CompuCell.Coordinates3DDouble(vx, vy, vz)
Parameters

_cell (CompuCell.CellG) – cell

Returns

instantaneous velocity of the cell at its center of mass

Return type

CompuCell.Coordinates3DDouble

unconditional_invariant_distance_vector_integer(p1, p2)¶

This function will calculate distance vector with integer coordinates between two Point3D points and make sure that the absolute values of the vector are smaller than 1/2 of the corresponding lattice dimension this way we simulate ‘invariance’ of distance assuming that periodic boundary conditions are in place. The reason we call it unconditional is because invariant distance this function computes assumes we have periodic boundary conditions in place irrespective if this is true or not. For some applications this function may be inappropriate. It is appropriate if the two points we are computing distance between are relatively close

Parameters

  • p1 – {list} position of first point

  • p2 – {list} position of second point

Returns

{ndarray} distance vector
